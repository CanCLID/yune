<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> runtime traits | User-provided paths and schema strings cross into RIME setup and schema selection. |
| Rust CLI -> C ABI function table | Unsafe pointer calls, struct initialization, and allocation/free ownership cross the FFI boundary. |
| Process -> process-wide RIME state | CLI command mutates global runtime/session state in `yune-rime-api`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01-01 | Tampering | `args.rs` runtime path flags | mitigate | Require explicit `--shared-data-dir` and `--user-data-dir` per D-02; reject missing values with corrective errors before calling setup. |
| T-01-01-02 | Elevation of Privilege | `rime_frontend.rs` FFI boundary | mitigate | Centralize all unsafe pointer/C string/ABI struct code in `rime_frontend.rs` per D-05 and initialize versioned `data_size` fields before every ABI call. |
| T-01-01-03 | Denial of Service | `rime_frontend.rs` process-wide RIME state | mitigate | Use a cleanup guard that destroys sessions, calls cleanup-all-sessions when available, and finalizes after success or failure per D-06. |
| T-01-01-04 | Tampering | `rime_frontend.rs` allocation ownership | mitigate | Pair every populated commit/context/status read with the matching free function before returning event data. |
| T-01-01-05 | Information Disclosure | CLI error output | mitigate | Error messages include problem and next action only; do not print absolute temp paths, pointers, memory addresses, or environment-derived data. |
</threat_model>
