<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Owned ABI event data -> stdout | Captured state crosses from runtime data into terminal/JSON output. |
| Fixture file -> replay engine | User-provided fixture content controls schema and sequence for ABI replay. |
| JSON/human renderers -> developer tooling | Output may be consumed by scripts and must remain deterministic and non-leaky. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | Information Disclosure | `transcript.rs` frontend JSON | mitigate | Serialize only owned ABI-visible fields from D-08; never serialize runtime paths, temp paths, pointers, timestamps, durations, process IDs, or environment values. |
| T-01-02-02 | Tampering | `fixture.rs` replay fixture parsing | mitigate | Extract only top-level `schema_id` and `sequence` strings needed for replay; reject malformed JSON string fields with corrective errors. |
| T-01-02-03 | Repudiation | `fixture.rs` mismatch output | mitigate | Include deterministic expected/actual transcript bodies in mismatch errors while omitting environment-specific context. |
| T-01-02-04 | Denial of Service | `render.rs` human output | accept | Transcript size is bounded by user-provided sequence length in this CLI surrogate; no long-running service or remote input exists in this phase. |
| T-01-02-05 | Spoofing | `render.rs` terminal text | mitigate | Use plain text labels only and prohibit ANSI escape/control sequences so output cannot disguise prompts or cursor state. |
</threat_model>
