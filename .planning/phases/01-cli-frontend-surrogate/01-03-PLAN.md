fn unique_temp_dir(label: &str) -> PathBuf {
    let nanos = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .expect("system clock should be after Unix epoch")
        .as_nanos();
    std::env::temp_dir().join(format!(
        "yune-rime-api-frontend-{label}-{}-{nanos}",
        std::process::id()
    ))
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Test temp runtime dirs -> RIME runtime | Integration tests provide filesystem paths to process-wide ABI runtime state. |
| Test harness -> global RIME state | Parallel tests can interfere through process-wide sessions and runtime globals. |
| Compatibility markers -> future planning | In-code ownership/comparison markers influence future executor behavior and must not misrepresent scope. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Denial of Service | `frontend_surrogate.rs` process-wide tests | mitigate | Serialize tests that touch RIME process-wide state with a mutex guard and always cleanup/finalize after each test path. |
| T-01-03-02 | Information Disclosure | transcript determinism tests | mitigate | Assert frontend JSON does not contain temp dirs, timestamps, process IDs, pointer-like addresses, or environment-derived data. |
| T-01-03-03 | Tampering | module ownership markers | mitigate | Keep comments tied to exact owning module behavior and explicit librime comparison targets; avoid broad claims that native frontend validation is complete. |
| T-01-03-04 | Repudiation | fixture replay tests | mitigate | Compare deterministic expected/actual output and keep mismatch bodies stable for reproducible failures. |
| T-01-03-05 | Elevation of Privilege | ABI test setup | mitigate | Use explicit temporary shared/user dirs and do not use arbitrary resource IDs or direct filesystem path identifiers beyond runtime roots. |
</threat_model>
