<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| schema YAML -> runtime data lookup | Untrusted schema dictionary/import/pack/vocabulary IDs cross into local file lookup. |
| compiled bytes -> core parser | Untrusted `.table.bin`, `.prism.bin`, and `.reverse.bin` bytes cross into bounded binary readers. |
| parser output -> translator install | Parsed dictionary entries become user-visible candidates. |
| compiled freshness -> runtime selection | Stale or malformed compiled artifacts can mask newer source dictionaries if selection is wrong. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | `schema_install.rs` resource lookup | mitigate | Validate dictionary, import, pack, vocabulary, table, prism, and reverse logical IDs with `validate_data_resource_id` before `selected_runtime_data_path`; add traversal/absolute/separator/drive-prefix tests in `dictionary_data.rs`. |
| T-04-01-02 | Denial of Service | `compiled_table.rs`, `compiled_prism.rs`, `compiled_reverse.rs` | mitigate | Use checked arithmetic and bounded string/array/relative-offset reads; reject huge counts and invalid lengths with structured errors; add malformed local byte tests. |
| T-04-01-03 | Information Disclosure | runtime data path resolution | mitigate | Keep schema-provided IDs as logical resource IDs only; never allow absolute paths or separators from schema IDs to select arbitrary files. |
| T-04-01-04 | Tampering | compiled freshness selection | mitigate | Compare compiled checksums to source-derived checksum where source exists; reject stale compiled data and test fallback to source. |
| T-04-01-05 | Repudiation | dictionary installation failures | mitigate | Replace silent `Option` miss with `DictionaryLoadOutcome::NoUsablePath` carrying dictionary ID and reason; test explicit failure when compiled and source paths both fail. |
| T-04-01-06 | Elevation of Privilege | binary parser placement | mitigate | Keep binary parsing in `yune-core` dictionary modules that accept bytes only; `schema_install.rs` chooses resources and never interprets raw binary layout. |
</threat_model>
