<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| schema YAML -> deployment artifact selection | Untrusted schema dictionary and pack IDs become candidate artifact names. |
| source/prebuilt bytes -> checksum and parser logic | Local data files influence rebuild decisions and generated compiled artifacts. |
| rebuild report -> runtime selection | Incorrect reporting can hide stale or missing artifacts. |
| deployment output -> runtime parser | Generated table/prism/reverse files are later consumed as compiled payloads. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | `deployment.rs` dictionary artifact paths | mitigate | Validate schema dictionary and pack IDs with `validate_data_resource_id` before constructing source/prebuilt/staging filenames; add traversal tests. |
| T-04-02-02 | Tampering | rebuild freshness decisions | mitigate | Use checksum-driven source/schema/pack comparisons instead of trusting mtimes alone; tests mutate source/schema/pack bytes and assert rebuilds. |
| T-04-02-03 | Repudiation | partial rebuild behavior | mitigate | Add `RimeDictRebuildExecutionReport` with explicit artifact statuses and tests asserting rebuilt/reused/prebuilt/missing states. |
| T-04-02-04 | Denial of Service | parsing existing compiled artifacts during deployment | mitigate | Reuse Plan 04-01 bounded parsers for metadata/payload validation and fail closed on malformed offsets/counts/lengths. |
| T-04-02-05 | Elevation of Privilege | rebuild implementation | mitigate | Do not shell out to external compilers or execute schema-provided paths; grep-gate against `Command::new`/`std::process` in deployment rebuild code. |
| T-04-02-06 | Information Disclosure | source-vs-prebuilt fallback | mitigate | Keep fallback inside validated runtime data roots and never treat dictionary IDs as arbitrary paths. |
</threat_model>
