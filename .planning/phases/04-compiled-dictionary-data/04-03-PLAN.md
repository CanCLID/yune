<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| advanced compiled sections -> core dictionary model | Stem, dict_settings, vocabulary, and encoder payload bytes become lookup behavior. |
| schema reverse lookup config -> dictionary selection | Reverse dictionary IDs and settings influence runtime translator output. |
| dictionary model -> translator output | Parsed advanced data becomes user-visible candidate/comment/reverse lookup output. |
| Phase 04 data -> Phase 5 excluded behavior | Advanced dictionary findings could accidentally introduce persistent learning or predictive ranking. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Tampering | compiled advanced section parsers | mitigate | Parse stems/settings/vocabulary/encoder sections with checked offsets/counts/lengths and structured unsupported-section errors; add malformed advanced fixture tests. |
| T-04-03-02 | Denial of Service | encoder and vocabulary payload handling | mitigate | Bound phrase generation and reject huge section counts/lengths before allocation; reuse Plan 04-01 parser limits. |
| T-04-03-03 | Information Disclosure | reverse dictionary resource lookup | mitigate | Continue validating reverse dictionary IDs with `validate_data_resource_id` before file lookup in `schema_install.rs`. |
| T-04-03-04 | Tampering | translator behavior from dict_settings | mitigate | Consume only recognized settings and preserve existing schema config precedence; unsupported settings become findings/errors instead of silent behavior changes. |
| T-04-03-05 | Elevation of Privilege | accidental Phase 5 scope creep | mitigate | Do not add LevelDB/userdb learning, plugins, predictive frequency, or AI ranking code; grep-gate for these terms and keep tests fixture-driven. |
| T-04-03-06 | Repudiation | unsupported advanced sections | mitigate | Structured unsupported findings include observed section role and target follow-up so unsupported data is not mistaken for compatibility. |
</threat_model>
