Plan 04-01 creates bounded compiled table/prism/reverse readers and schema-install compiled/source fallback.

Plan 04-03 promotes dictionary data contracts and advanced payload flow. Build correction/tolerance support on those contracts rather than creating separate translator-only structs.

Existing lookup path in `StaticTableTranslator` compares lookup code to entry code with completion support:
```rust
fn matches_lookup_code(&self, entry_code: &str, lookup_code: &str) -> bool {
    entry_code == lookup_code
        || (self.enable_completion && !lookup_code.is_empty() && entry_code.starts_with(lookup_code))
}
```

Correction/tolerance integration should extend lookup inputs around this function while preserving exact-match and completion behavior for existing tests.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| correction/tolerance compiled sections -> parser | Untrusted prism/table payload sections define alternate lookup behavior. |
| parsed correction/tolerance data -> translator | Parsed data changes which candidates are returned for user input. |
| unsupported binary observations -> compatibility claims | Unknown sections can create false compatibility if silently ignored. |
| schema-loaded tests -> external oracle assumptions | Librime-derived observations inform fixtures but normal tests must not depend on external artifacts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Tampering | correction/tolerance payload parsers | mitigate | Use checked offsets/counts/lengths/UTF-8 parsing and reject malformed local bytes with structured errors. |
| T-04-04-02 | Denial of Service | tolerance expansion | mitigate | Bound tolerance rule counts and expansion lists; reject huge allocations before lookup; test huge-count payloads. |
| T-04-04-03 | Tampering | translator candidate ordering | mitigate | Evaluate exact matches before correction/tolerance expansion and assert deterministic source/compiled ordering parity. |
| T-04-04-04 | Repudiation | unsupported correction/tolerance sections | mitigate | Return structured unsupported findings with observed role and target follow-up; add tests that unsupported sections are not silent no-ops. |
| T-04-04-05 | Elevation of Privilege | external compiler/oracle usage | mitigate | Normal tests use local Rust-generated fixture bytes only; grep-gate against external command execution. |
| T-04-04-06 | Information Disclosure | correction/tolerance lookup data | accept | Data is local dictionary metadata and not PII in this phase; no external transmission is introduced. |
</threat_model>
