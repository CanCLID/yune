<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| C ABI caller to `yune-rime-api/src/userdb.rs` | Untrusted frontend pointers and dictionary names enter exported ABI functions. |
| Snapshot/import file to userdb store | Untrusted snapshot metadata and records can select destination state and mutate persistence. |
| Runtime user data directory to file-backed store | Interrupted writes, stale temp files, and legacy artifacts can affect committed userdb state. |
| Sync input to merge engine | Large or conflicting userdb snapshots can exhaust memory or corrupt deterministic merge state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01-01 | Tampering/Elevation | `validate_user_dict_name` and userdb path construction | mitigate | Tasks 1-2 must call `validate_user_dict_name` before every dictionary/snapshot destination path; tests reject traversal, separators, suffix smuggling, Windows drive prefixes, and `~`. |
| T-05-01-02 | Tampering | `snapshot.rs` restore/import | mitigate | Task 2 validates snapshot `db_name` and records into a temp transaction first; malformed snapshots return failure and leave existing state unchanged. |
| T-05-01-03 | Denial of Service/Tampering | `file_store.rs` commit | mitigate | Task 1 implements temp-file plus atomic rename and Task 2 recovery/rollback; tests simulate interrupted write and verify prior committed state is readable. |
| T-05-01-04 | Denial of Service | `sync.rs` merge | mitigate | Task 2 replaces full destination+snapshot string and `HashSet` line merge with ordered keyed record merge; grep gate blocks reintroducing full-file line dedupe. |
| T-05-01-05 | Spoofing/Tampering | FFI pointers in `userdb.rs` | mitigate | Task 1 keeps pointer validation and exported ABI surface in `userdb.rs`; null/invalid pointer tests stay in ABI-facing test modules. |
| T-05-01-06 | Information Disclosure | Snapshot/export files | accept | Snapshot content is local user dictionary data under runtime user data/sync directories; no network export is introduced in this plan. |
</threat_model>
