<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dynamic library -> test harness | The test harness loads the Cargo-built `yune-rime-api` cdylib and dereferences the returned `RimeApi` function table. |
| Host-shaped scenario -> RIME ABI | The scenario passes raw pointers, C strings, session IDs, and caller-owned structs across the C ABI boundary. |
| ABI allocated data -> caller free functions | `get_context`, `get_status`, and `get_commit` populate caller-owned structs whose allocated fields must be released by matching `free_*` functions. |
| Runtime paths/resource IDs -> filesystem | Setup paths are local temp directories, while schema/config/userdb names must remain logical IDs before filesystem joins. |
| Trace fixture -> repository | Captured traces are committed fixtures and must not contain local paths, timestamps, pointer addresses, process IDs, or personal data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | Tampering | FFI free pairing in `frontend_hosts/native.rs` | mitigate | Record every successful `get_context`, `get_status`, and `get_commit` event with the exact matching `free_context`, `free_status`, or `free_commit` event before discarding the struct; fail the test if a free is missing or mismatched. |
| T-06-01-02 | Denial of Service | Null function pointers in the dynamic `RimeApi` table | mitigate | Use required-function helpers that record a validation blocker/mismatch for missing entries and never call a null pointer. |
| T-06-01-03 | Elevation of Privilege | Stale sessions after destroy, cleanup, finalize, or repeated initialize/finalize | mitigate | Add deterministic stale-session trace checks proving `find_session`/`destroy_session` reject stale IDs and new session creation after finalize behaves as expected. |
| T-06-01-04 | Information Disclosure | `fixtures/frontend-traces/native-host-lifecycle.json` | mitigate | Sanitize trace data to omit temp paths, user home paths, process IDs, timestamps, raw pointer addresses, Cargo target paths, and real user dictionary contents; include automated assertions that these tokens are absent. |
| T-06-01-05 | Tampering | Logical resource IDs used by schema/config/userdb validation inputs | mitigate | Use logical IDs such as `dynamic_schema` only; reject or avoid path traversal, absolute paths, platform separators, and other filesystem syntax in trace scenarios. |
| T-06-01-06 | Repudiation | Frontend-observed mismatch capture format | mitigate | Store target, scenario, ordered call sequence, expected behavior, observed behavior, classification, and reproduction status in the trace/mismatch contract so later fixes have auditable pre-fix evidence. |
</threat_model>
