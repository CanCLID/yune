<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External TypeDuck-Web source/docs -> Yune tests | The executor may inspect external wrapper shape, but only minimized call-sequence knowledge enters Yune; no vendored untrusted frontend code is copied into the repository. |
| Browser/WebAssembly wrapper model -> native RIME ABI | Browser worker, IDBFS, and Emscripten lifecycle assumptions are modeled at the ABI call boundary and must not be mistaken for native dynamic-loader or OS input-context coverage. |
| Rust tests -> C ABI pointers | Test code crosses into `RimeApi` function pointers and caller-owned C structs; every successful allocation-bearing read must be paired with the exact free/destroy API. |
| Trace generation -> repository fixtures/docs | Validation traces can accidentally capture local filesystem paths, process data, raw pointers, timestamps, or personal user dictionary data; checked-in artifacts must be synthetic and sanitized. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Tampering | `crates/yune-rime-api/tests/frontend_hosts/typeduck_web.rs` external wrapper model | mitigate | Do not vendor TypeDuck-Web code per D-09; encode only a minimized ABI call sequence and document inspected external wrapper files by name in `docs/frontend-validation/typeduck-web.md`. |
| T-06-02-02 | Information Disclosure | `fixtures/frontend-traces/typeduck-web-basic.json` and `docs/frontend-validation/typeduck-web.md` | mitigate | Use synthetic temp fixtures only; tests and JSON checks must reject local absolute paths, `/tmp`, raw pointers, timestamps, process IDs, Cargo env vars, and personal user data before artifacts are accepted. |
| T-06-02-03 | Denial of Service | Browser/WebAssembly interpretation in validation results | mitigate | Classify Emscripten worker lifecycle, IDBFS persistence, and unavailable native dynamic loading as browser/WASM limits per D-10 so they do not block native ABI work or misdirect fixes. |
| T-06-02-04 | Elevation of Privilege | FFI allocation ownership in TypeDuck-Web-shaped test | mitigate | Pair every successful `get_context`, `get_status`, `get_commit`, iterator/list, and levers allocation with the matching `free_*` or destroy call; assert pairing in tests rather than relying on comments. |
| T-06-02-05 | Spoofing | Function-table entry assumptions | mitigate | Required `RimeApi` entries must be checked through the 06-01 missing-function/mismatch format; missing entries become reproducible blockers, not unchecked calls. |
| T-06-02-06 | Repudiation | Frontend-observed ABI/runtime mismatches | mitigate | For each observed mismatch, record target, call sequence, expected behavior, observed behavior, classification, and artifact/test reference before proposing or applying any fix, satisfying `FRONTEND-VALIDATION-05`. |
| T-06-02-07 | Tampering | Resource/config/userdb IDs in validation scenario | mitigate | Use logical IDs only for schema/config/userdb names; do not introduce path traversal, absolute paths, or platform separators into IDs while modeling browser persistence. |
</threat_model>
