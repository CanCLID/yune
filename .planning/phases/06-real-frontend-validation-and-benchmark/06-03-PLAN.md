<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| native frontend notes -> repository docs/fixtures | Optional Squirrel, ibus, or fcitx validation can expose local machine paths, usernames, toolchain paths, environment variables, or personal input data unless sanitized before committing artifacts. |
| source-modeled frontend fixture -> RIME ABI function table | Tests drive C ABI pointers, function tables, sessions, callbacks, and caller-owned structs where null checks and free pairing are required for safety. |
| optional external frontend/toolchain -> normal regression suite | Real Squirrel, ibus, fcitx5, Xcode, Linux desktop daemons, or Emscripten-style toolchains are outside the project test trust boundary and must not become mandatory for deterministic regressions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03-01 | I | `docs/frontend-validation/squirrel-macos.md`, `docs/frontend-validation/linux-frontends.md`, `fixtures/frontend-traces/squirrel-lifecycle.json` | mitigate | Sanitize blocker notes and trace fixtures: omit absolute personal paths, usernames, raw pointers, timestamps, process IDs, Cargo target paths, and real user dictionary/input contents; use synthetic temp/runtime names only. |
| T-06-03-02 | T | `fixtures/frontend-traces/squirrel-lifecycle.json` | mitigate | Validate JSON with `python3 -m json.tool`; include stable fields for target, call_sequence, expected_behavior, observed_behavior, blocker_or_gap, and reproduction_status so fixture edits remain reviewable and deterministic. |
| T-06-03-03 | D | `crates/yune-rime-api/tests/frontend_hosts/native_frontends.rs` | mitigate | Serialize tests touching process-wide runtime/session/notification state with the established test guard pattern; ensure cleanup/finalize runs even when assertions fail where helper structure permits. |
