<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser caller -> TypeScript wrapper | Caller passes paths, schema IDs, key events, candidate indices, and lifecycle calls. |
| TypeScript wrapper -> Emscripten Module | Wrapper binds raw C/WASM symbols and receives numeric pointers. |
| Emscripten Module -> Rust adapter | Rust owns state and response allocation; TypeScript must not double-free or leak pointers. |
| Browser host filesystem -> adapter paths | Phase 8 accepts paths but must not mount, preload, or sync virtual filesystems. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-01-01 | Tampering | Adapter symbol binding | high | mitigate | Bind exactly the 11 canonical `yune_typeduck_*` symbols from `scripts/typeduck-exports.txt`; throw `TypeDuckBindingError` for missing/non-callable exports. |
| T-08-01-02 | Denial of Service | Response pointer ownership | high | mitigate | Centralize parsing/freeing in `readTypeDuckResponse` and free every non-null response pointer in `finally`. |
| T-08-01-03 | Tampering | State pointer lifecycle | high | mitigate | Keep state pointers private, make `cleanup()` idempotent, zero state pointer after cleanup, and reject operations after cleanup. |
| T-08-01-04 | Tampering | Keyboard mapping | medium | mitigate | Use explicit `event.key` mapping to RIME constants and throw on unsupported keys; do not use deprecated `keyCode`. |
| T-08-01-05 | Information Disclosure | Browser filesystem/persistence | medium | mitigate | Accept path strings only; do not implement MEMFS/IDBFS mount, asset preload, persistence sync, remote calls, or AI provider behavior. |
</threat_model>
