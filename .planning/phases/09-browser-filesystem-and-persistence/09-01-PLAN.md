<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser host caller -> TypeScript filesystem helper | Caller-provided paths, schemaId, dictionaryId, and asset contents enter the package helper. |
| TypeScript filesystem helper -> Emscripten FS | Helper writes directories and files into the virtual filesystem consumed by native init. |
| TypeScript preflight -> native adapter init | Preflight must match native guard so missing assets fail before process-global runtime startup. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | `isTypeDuckLogicalId` / path joins | mitigate | Reject schema and dictionary IDs outside nonempty `[A-Za-z0-9_-]+` before constructing virtual paths per D-07. |
| T-09-02 | Tampering | `prepareTypeDuckFilesystem` | mitigate | Require explicit caller-provided default/schema/dictionary asset contents and never generate placeholder YAML per D-05/D-06. |
| T-09-03 | Denial of Service | Native process-global init path | mitigate | Run `assertTypeDuckAssetsReady` before `TypeDuckRuntime.init` so missing files fail before starting the process-global service per D-03/D-06. |
| T-09-04 | Elevation of Privilege | `EmscriptenTypeDuckModule` boundary | mitigate | Keep filesystem interface separate from native binding interface and avoid native export expansion per D-01. |
</threat_model>
