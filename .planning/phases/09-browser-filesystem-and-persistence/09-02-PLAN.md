<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TypeScript helper -> Emscripten persistence backend | Helper invokes host-provided `mount` and `syncfs` callbacks. |
| Native runtime mutation -> browser persistent storage | Runtime deploy/customize/userdb-changing flows update in-memory virtual files that must be synced by the host. |
| Callback error -> caller recovery policy | Emscripten sync errors cross into TypeScript error handling and recovery decisions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-05 | Tampering | `syncTypeDuckFilesystem` | mitigate | Encode sync direction names so before-init uses `syncfs(true)` and after-mutation uses `syncfs(false)` per D-08. |
| T-09-06 | Repudiation | Sync callback error path | mitigate | Surface callback failures as deterministic `TypeDuckFilesystemError` instead of swallowing them per D-10. |
| T-09-07 | Tampering | `deployAndSync` / `customizeAndSync` | mitigate | Persist after public runtime mutation calls and document/verify that failed sync leaves in-memory changes requiring retry per D-11. |
| T-09-08 | Elevation of Privilege | Native export surface | accept | No new native exports are added; userdb mutation visibility remains a documented host sync boundary per D-08 and D-11. |
</threat_model>
