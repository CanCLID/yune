<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TypeDuck-Web app events -> Yune runtime adapter | Browser keyboard/candidate/deploy/customize events enter the Yune TypeScript bridge. |
| TypeDuck-Web assets -> Emscripten filesystem | App-owned YAML assets are written into MEMFS/IDBFS paths consumed by native runtime. |
| TypeDuck-Web patch -> upstream source | Patch changes third-party source files and may alter app behavior. |
| Browser persistence -> runtime init | Persisted user data is loaded before native init and flushed after mutations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Tampering | Patch application | mitigate | Keep patch hunks limited to documented seam/config files and verify no dependency/build-output hunks per D-03. |
| T-10-02-02 | Tampering | YAML assets | mitigate | Require explicit TypeDuck-Web-owned `default.yaml`, schema YAML, and dictionary YAML; grep-gate fallback/dummy/placeholder assets per D-06. |
| T-10-02-03 | Denial of Service | Runtime lifecycle | mitigate | Enforce one active `TypeDuckRuntime` per Emscripten Module and deterministic cleanup in adapter per D-05. |
| T-10-02-04 | Information Disclosure | Browser persistence | mitigate | Use `syncFromPersistenceBeforeInit` and post-mutation sync helpers explicitly; do not add network persistence or silent fallback behavior per D-04/D-10. |
| T-10-02-05 | Elevation of Privilege | Yune adapter widening | mitigate | Widen Yune runtime only for the smallest proven blocker and document mismatch before change per D-07. |
| T-10-02-06 | Spoofing | Accidental original librime path | mitigate | Patch must route seam through `@yune-ime/typeduck-runtime`/`TypeDuckRuntime`; findings must record any remaining original librime/WASM calls. |
</threat_model>
