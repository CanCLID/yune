<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser automation -> TypeDuck-Web UI | E2E runner sends keyboard/mouse actions into the real app. |
| TypeDuck-Web UI -> Yune runtime | Patched app routes input and candidate actions into Yune runtime helpers. |
| Browser persistence -> reload/reinitialize | Persisted IDBFS/equivalent data crosses browser session boundaries. |
| Test assets -> virtual filesystem | App-owned YAML assets are loaded into Emscripten FS for native runtime consumption. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03-01 | Spoofing | Browser validation target | mitigate | Spec must target the patched upstream TypeDuck-Web app URL, not package-local fake modules per D-08. |
| T-10-03-02 | Tampering | Test assets | mitigate | Require explicit app-owned YAML assets and grep-gate fallback/dummy/placeholder schema/dictionary wording per D-06. |
| T-10-03-03 | Repudiation | Skipped browser tooling | mitigate | Record command, missing dependency, and fallback evidence for any browser/local blocker; missing browser tooling is BLOCKED, not PASS per D-09. |
| T-10-03-04 | Information Disclosure | Browser local persistence | mitigate | Validate persistence only through local browser storage sync/reload; do not add network storage or remote fallback behavior. |
| T-10-03-05 | Denial of Service | Persistence ordering | mitigate | Require sync-from before init, sync-to after mutation, and reload/reinitialize proof per D-10. |
| T-10-03-06 | Tampering | Accidental network/fallback behavior | mitigate | Findings must record unexpected network calls or fallback asset generation as blockers before recommendation. |
</threat_model>
