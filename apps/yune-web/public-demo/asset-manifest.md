# yune-web Asset Manifest

M31 public deployment ships one public schema surface: `jyut6ping3_mobile`.
The checked-in JSON manifest is the authoritative deploy list.

Included asset groups:

- Yune/Yune web runtime shell generated by the public build: app chunks,
  `worker.js`, `yune-web.js`, and `yune-web.wasm`.
- Active schema assets for `jyut6ping3_mobile`.
- `jyut6ping3` dictionary and compiled table/reverse/prism files.
- `jyut6ping3_scolar` lookup assets used by the current dictionary lookup
  filter.
- Luna Pinyin assets required by the current mobile schema dependency and
  reverse lookup path.
- OpenCC `hk2s` files used by the public Simplified Chinese output standard.

Pruned from the public deploy:

- Cangjie 3/5 selectable schemas and dictionaries.
- Loengfan schemas/dictionary.
- Side mobile layouts such as longpress and 10-key schemas.
- Internal dogfood-only schema switcher controls.

Pinned public schema payload bytes from `schema-asset-manifest.json`: 30 files,
29,414,157 bytes. The final deployable `dist/` package contains 42 files and
32,148,337 bytes including app JS/CSS, `worker.js`, `yune-web.js`, and
`yune-web.wasm`.

Measured caveat: the current public schema still boots Luna and scholar lookup
assets because the public `jyut6ping3_mobile` schema depends on them. M31 excludes
non-public schema families and content-addresses the public asset cache, but it
does not claim a full reverse-lookup dependency deferral.
