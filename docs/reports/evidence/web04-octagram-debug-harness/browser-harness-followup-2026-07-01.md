# WEB-04 Browser Harness Follow-Up

Date: 2026-07-01.

Verdict: **complete after review follow-up**. Tasks 1-5 are implemented and
verified in the app/browser lane. The follow-up fixes the review findings around
default-off safety, diagnostic wording, non-vacuous browser evidence, fetch
atomicity, and public-demo hygiene.

The first browser closeout used the shorthand "loaded" too broadly. The final
contract is narrower:

- `delivered=true` means the browser fetched and checksum-verified the pinned
  `.gram`, then attached it through `extraSharedAssets`.
- Engine use is proven by the Playwright ranking rows, not by the delivery flag
  alone.
- `memoryDeltaBytes` is a schema-select/redeploy WASM heap high-water delta. It
  includes JS fetch/cache, MEMFS delivery, and engine state; it is not model-only
  attribution.
